메타태그 테스트 페이지

📋 테스트 방법

  1. 개발 서버에서 프로모션 뷰 페이지로 이동
  2. F12를 눌러 개발자 도구 열기
  3. Elements 탭에서 <head> 섹션 확인
  4. 아래 예상 값들과 비교

🎯 예상되는 메타태그 값들

1. Title 태그

<title>렌탈 톡톡 월간 혜택 [웹 카탈로그]</title>

설명: 시스템 설정의 defaultTitle이 우선 사용됩니다.

2. Open Graph Title

<meta property="og:title" content="렌탈 톡톡 월간 혜택 [웹 카탈로그]" />

설명: 카카오톡, 페이스북 등에서 보이는 제목입니다.

3. Description

<meta name="description" content="최신 렌탈 정보와 프로모션을 확인하세요" />

설명: 시스템 설정의 defaultDescription이 사용됩니다.

4. Open Graph Description

<meta property="og:description" content="최신 렌탈 정보와 프로모션을 확인하세요" />

5. Site Name

<meta property="og:site_name" content="렌탈톡톡" />

🔍 확인 방법

개발자 도구에서 확인:

  1. 프로모션 뷰 페이지 방문
  2. F12 → Elements 탭
  3. <head> 태그 펼치기
  4. 위의 메타태그들이 올바른 값으로 설정되어 있는지 확인

브라우저 탭에서 확인:

  • 브라우저 탭의 제목이 "렌탈 톡톡 월간 혜택 [웹 카탈로그]"인지 확인

🚀 실제 링크 미리보기 테스트

실제 링크 미리보기를 테스트하려면:

  1. 빌드 후 배포: npm run build → Netlify/Vercel 등에 배포
  2. ngrok 사용: 로컬 서버를 임시로 외부에 노출
  3. 온라인 테스트 도구: 배포된 URL로 Facebook Open Graph Debugger 등 사용
